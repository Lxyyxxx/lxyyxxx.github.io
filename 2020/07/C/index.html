# 和C的区别

  1. 头文件不带.h
  2. 有命名空间，使用系统库时使用命名空间using namespace std;
  3. main函数为int类型，程序正常执行return 0;
  4. 添加了class，默认对象为private，封装，信息隐蔽
  5. 在struct内可以定义成员函数，public
  6. 面向对象而不是面向过程（过程：程序=算法+数据结构程序=算法+数据结构程序=算法+数据结构；对象：程序=对象+对象+…程序=对象+对象+\dots程序=对象+对象+…
  7. string类型处理字符串
  8. 有STL
  9. 可以使用流输入输出
  10. C要求变量定义必须在所有执行语句前，C++只要求在第一次使用该变量前进行定义
  11. C++中有bool型
  12. C定义函数时可以不指定函数类型，但C++必须指定

# 变量

在程序的运行期间可以改变的量

// 变量类型：int
// 变量名：a，代表内存中的一个存储单元
// 变量值：3，变量名对应的值
int a=3;

合法的标识符（实体名字的有效序列）：

  • 由字母、数字下划线组成，第一个字符必须是字母或下划线
  • 不能与关键字、系统函数名、类名相同

其他需要注意的：

  • 区分大小写，一般变量名用小写字母表示，常习惯在变量前加一个字母表示变量类型
  • 强制定义：先定义（声明），后使用
  • 在同一作用域内，不能被重复定义

强制定义的目的：

  1. 保证程序中变量名使用正确
  2. 编译时就可以分配存储单元
  3. 编译时就可以检查运算是否合法

变量初始化：

  • 不是在编译阶段完成的，实在执行本函数时赋值的，int a=3;等价于int a;a=3;
  • 在声明语句中，必须int a=1,b=1,c=1;
  • 在执行语句中，可以int a,b,c=1;a=b=c;

# 内置函数

inline
在编译时将所调用的函数代码直接嵌入到主调函数中，而不是将流程转出去，可以减少时间的开销（一般情况下，转到被调用函数前需要记下当前执行指令的地址，以及保存现场），但会增加目标文件中main函数的长度，一般只将规模很小（5个语句以下）且频繁使用的函数声明为内置函数

# 变量的值作为函数参数

无法通过形参修改实参，完成不了修改多个值的任务

# 指针作为函数参数

将变量的地址传送给被调用函数的形参

void swap(int*, int*);
int main() {
    int a = 10, b = 20;
    int* pa = &a, * pb = &b;
    cout << pa << " " << pb << endl;    // 00F9F89C 00F9F890
    cout << a << " " << b << endl;      // 10 20
    swap(pa, pb);
    cout << pa << " " << pb << endl;    // 00F9F89C 00F9F890
    cout << a << " " << b << endl;      // 20 10
    return 0;
}
void swap(int* p1, int* p2) {
    // p1, p2不变，*p1，*p2改变
    int t = *p1;
    *p1 = *p2;    // 不是 p1 = p2
    *p2 = t;
}

说明：

  1. 通过调用函数使变量的值发生改变，在主调函数中使用这些改变了的值
  2. 调用函数时不会改变实参指针的值，但可以改变实参指针变量指向的值

# 引用作为函数参数

传递变量的指针可以做到形参指向实参的变量单元，从而改变实参的值，但仍然是值传递方式，只是传的值是变量的地址而不是变量的值
可以通过传递变量的别名，通过修改别名的值从而修改实参的值

void swap(int&, int&);
int main() {
    int a = 10, b = 20;
    cout << a << " " << b << endl; // 10 20
    swap(a, b);
    cout << a << " " << b << endl; // 20 10
    return 0;
}
void swap(int &x, int&y) {    // x是a的别名，y是b的别名
    int tmp = x;
    x = y;
    y = tmp;
}

说明：

  1. 引用作为函数参数，不用开辟内存空间，但指针作为函数参数时需要开辟内存空间
  2. 实参前不能加&，参考int &b=a;写实参

# 作用域：局部变量和全局变量

每个变量都有其有效的作用区域，在作用域外不能访问变量
局部变量：在局部区域（函数、复合语句）内定义的变量，在这些区域内才能使用它们
全局变量：在函数之外定义的变量（外部变量），有效范围是定义变量的位置到本源文件结束

说明：

  1. 主函数中定义的变量也只在主函数中有效，主函数也不能使用其他函数中定义的变量
  2. 不同含函数中可以使用同名的变量，它们代表不同的对象，在内存中占不同的存储单元，互不干扰
  3. 可以在一个函数内的复合语句中定义变量，这些变量只在本复合语句中有效，这种复合语句称为分程序或程序块
  4. 形参也是局部变量
  5. 函数声明中出现的参数名，其作用域只在本行的括号内（编译器忽略，调用时也没有分配存储单元）
  6. 全局变量的作用是增加函数间数据联系的渠道
  7. 建议不在必要时不要使用全局变量，因为
    • 全局变量在程序的全部执行过程中都要占用存储单元
    • 降低了函数的通用性和可靠性，除了移动函数还要移动全局变量，同名时会出现问题，不符合模块内聚性强，与其他模块的耦合性弱的原则
    • 降低程序清晰性
  8. 同一个源文件中，全局变量的局部变量同名，则在局部变量的作用范围内，全局变量被屏蔽

不同的作用域：

  1. 文件作用域（全局）
  2. 函数作用域
  3. 块作用域
  4. 函数原型作用域

任何以标识符代表的实体都有作用域

# 存储期：动态存储和静态存储

变量在内存中的存在期间，由变量的存储方式决定
内存中供用户使用的存储空间可以分为三部分：

  1. 程序区
  2. 静态存储区
  3. 动态存储区

全局变量全部存在静态存储区中，程序执行时给全局变量分配存储单元，程序执行完毕后就释放这些空间，程序执行过程中占固定的存储单元
动态存储区中存放：

  1. 函数形参
  2. 函数中的自动变量（未加static声明的局部变量）
  3. 函数调用时的现场保护和返回地址

调用时分配，调用完毕时释放，多次调用多次分配，每次分配的存储地址可能不同

# auto

自动变量，即自动分配/释放存储空间，程序中大多数变量属于自动变量，auto可省略，默认指定为自动变量

# static 静态局部变量

函数的局部变量在函数调用后不消失而保留原值，即其占用的存储空间不释放，下一次调用时，保留上一次函数调用结束时的值

int f(int);
int main() {
    int a = 2;
    for (int i = 0; i < 3; i++)
        cout << f(a) << endl;
    return 0;
}
int f(int a) {
    int b = 0;
    static int c = 3;
    b++;
    c++;
    return a + b + c;
}
// ===============================
// i    a    b    c    f(a)
// 0    2    1    4    7
// 1    2    1    5    8
// 2    2    1    6    9

说明：

  1. 静态局部变量在静态存储区内分配存储单元，整个运行期间都不释放
  2. 只赋一次初值，在编译时，程序运行时已经有初值
  3. 对于静态局部变量，定义时不赋初值，编译时自动赋0或空字符。对于自动变量，不赋初值，值是一个不确定的值，因为每次分配的分配单元中的值是不确定的
  4. 其他函数不能引用静态局部变量

使用静态局部变量的情况：

  1. 需要保留上一次调用结束时的值
  2. 初始化后，变量只被引用而不改变其值

# 数组 []

有序数据的集合

# 定义

// 一维数组
类型标识符 数组名[常量表达式];

// 二维数组
类型标识符 数组名[常量表达式][常量表达式];

说明：

  1. 数组名遵循标识符定名规则
  2. 常量表达式表示下标值
  3. 常量表达式代表元素的个数，即数组长度
  4. 常量表达式可包括常量、常变量和符号常量，不能包含变量
  5. 按行存放，内存中先放行的元素，再放列的元素，[row][col]
  6. 允许使用多维数组，第一维可以不指定，但后面的必须指定。因为要分配连续的内存空间，必须知道每行有多少个元素才嫩保证连续
int n;
cin>>n;
int a[n];    // 不合法
// ======================
const int n=5;
int a[n];    // 合法

如果int *p=&a[0]，则：

  1. p+i和a+i就是a[i]的地址，它们指向数组第i个元素
  2. a代表首元素地址，a+i计算方法同p+i，实际地址为a+i*d，d=sizeof(int)
  3. *(p+1)或*(a+1)是p+i或a+i所指向的数组元素，即a[i]
  4. []是变址运算符，a[i]求解过程相当于a+i

# 引用 &

为变量取一个别名

int a;

说明：

  • &是引用声明符
  • 声明b是a的引用，可以理解为：使变量b具有a的地址

# 结构体 struct

# 声明

struct 结构体类型名{
    成员表列(类型名 成员名;) // 必须对每个成员都进行声明
};    // 不要忘记了分号

说明：

  1. 每一个成员也称为结构体中的一个域
  2. 成员表列由被称作域表
  3. C中结构体成员只能是数据，C++扩充后可以是函数

# 定义

// 1. 先声明再定义
结构体类型名 结构体变量名;
// 2. 声明时同时指定
struct 结构体类型名{
    成员表列
}变量名表列;
// 3. 直接定义
 struct {     // 注意这里没有结构体类型名
    成员表列
}变量名表列;

说明：

  1. 每个结构体类型都有自己的结构，即结构不一定相同
  2. 只能对结构体变量中的成员赋值，而不能对结构体类型赋值
  3. 编译时，不会为类型分配存储空间，只为变量分配空间
  4. 结构体中的成员可以单独引用，作用和地位相当于普通变量
  5. 成员也可以是一个结构体变量
  6. 成员名可以与程序中的变量名相同

# 初始化

// 1. 结构体变量
Student stu1,stu2;
stu1 = stu2;
// 2. 结构体成员变量
stu1.num = 100;

说明：

  1. .是成员运算符
  2. 成员必须逐级引用
  3. 不能将结构体变量直接输入输出，只能直接输入输出结构体成员变量
  4. 可以引用结构体变量或结构体成员变量的地址

# 析构函数

以下几种情况程序会执行析构函数：

  1. 函数调用结束对象被释放前，自动执行析构函数
  2. main函数结束或调用exit结束程序时，调用static局部对象的析构函数
  3. 全局对象在程序的流程离开其作用域时，调用该全局对象的析构函数
  4. new建立对象，在delete时，先调用该对象的析构函数
class Time {
    int hour, minute, second;
public:
    // 默认构造函数
    Time() :hour(0), minute(0), second(0) {}
    // 重载构造函数
    Time(int, int, int);
    // 析构函数
    ~Time() {
        print();
        cout << "~Time()\n";
    }
    void print() {
        cout << hour << " " << minute << " " << second << endl;
    }
};
Time::Time(int h, int m = 59, int s = 59) {
    hour = h;
    minute = m;
    second = s;
}
int main() {
    Time t1;
    t1.print();
    Time t2{23};
    t2.print();
    return 0;
}
// ================
// 0 0 0
// 23 59 59
// 23 59 59
// ~Time()
// 0 0 0
// ~Time()

注意：

  1. 析构函数的作用是在撤销对象占用的内存前完成一些清理工作，并不是删除对象
  2. 析构函数没有函数参数，不能被重载
  3. 一个类有且仅有一个析构函数

# 调用构造函数和析构函数的顺序

先构造的后析构，后构造的先析构，相当于一个栈，先进后出

对象作用域 调用构造函数 调用析构函数
全局 文件所有函数执行之前 main执行完毕或调用exit函数
局部 调用函数建立对象时 函数调用结束对象释放时
static局部 第一次调用函数建立对象时 main执行完毕或调用exit函数

# 继承与派生

在C++中，通过继承实现可重用性
在原有的基类/父类的基础上建立一个新的类，称为派生类/子类
派生可以一代又一代地派生下去，形成类的继承结构
单继承：一个派生类只从一个基类派生
多重继承：一个派生类可以从多个基类派生出来
派生类时基类的具体化，基类是派生类的抽象

# 声明

继承方式包括：public（公用继承/公有基类）、private（私有继承/私有基类）、protected（保护继承/保护基类），默认private

# 工作流程

  1. 从基类接收成员，把基类**全部成员（不包含构造函数和析构函数）**全部接收过来，缺点是会造成数据冗余
  2. 调整从基类接受的成员，根据继承方式更改派生类中成员的访问属性。可以在派生类中声明一个同名的成员，如果函数的参数表不同，是重载，其他情况均是覆盖
  3. 在声明派生类时增加的成员
  4. 定义派生类自己的构造函数和析构函数

# 多态性与虚函数

不同功能的函数可以用同一个函数名，即调用同一个函数名的函数，但执行的并不是同一函数
静态多态性：重载，编译时就能决定调用哪个函数
动态多态性：虚函数，程序运行时才动态地确定操作所针对的对象
虚函数的作用：允许在派生类中重新定义与基类同名的函数，并可以通过基类指针或引用来访问基类和派生类中的同名函数
未引入虚函数：

class A {
protected:
    int num;
public:
    A() :num(-1) {}
    A(int n) :num(n) {}
    void print() {
        cout << "num:" << num << endl;
    }
};
class B :public A {
    int age;
public:
    B() :age(-2) {}
    B(int a) :age(a) {}
    B(int n, int a) :A(n), age(a) {}
    void print() {
        cout << "num:" << num << " age:" << age << endl;
    }
};
int main() {
    A a;
    B b(6, 66);
    A* p = &a;
    p->print();
    p = &b;
    p->print();    // 调用的是A的print函数，输出的是B中继承A的数据成员
    return 0;
}

*/

把基类的print设为虚函数：

class A {
protected:
    int num;
public:
    A() :num(-1) {}
    A(int n) :num(n) {}
    virtual void print() {    // 设为虚函数
        cout << "num:" << num << endl;
    }
};
class B :public A {
    int age;
public:
    B() :age(-2) {}
    B(int a) :age(a) {}
    B(int n, int a) :A(n), age(a) {}
    void print() {
        cout << "num:" << num << " age:" << age << endl;
    }
};
int main() {
    A a;
    B b(6, 66);
    // 通过指针 
    A* p = &a;
    p->print();
    p = &b;
    p->print();
    // 或通过引用
    A &r = b;
    r.print();
    return 0;
}
/*
num:-1
num:6 age:66
num:6 age:66
*/

虚函数使用方法：

  1. 在基类用virtual声明成员函数为虚函数
  2. 在派生类中重新定义此函数，要求函数名、函数类型、函数参数个数和类型全部与基类的虚函数相同，并重新定义函数体
  3. 定义一个指向基类对象的指针变量，并使它指向同一类族中需要调用该函数的对象
  4. 通过该指针调用此虚函数，此时调用的就是指针变量指向的对象的同名函数

说明：

  1. 当一个成员函数被声明为虚函数后，其派生类中的该函数都自动成为虚函数
  2. 如果派生类中没有对基类的虚函数重新定义，则派生类简单地继承其直接基类的虚函数

# 在什么情况下应当声明虚函数

使用时的注意事项：

  1. 只能用virtual声明类的成员函数让它成为虚函数，而不能将类外的普通函数声明为虚函数
  2. 虚函数只能用于类的继承层次结构中
  3. 一个成员函数被声明为虚函数后，在同一类族中的类就不能再定义一个非virtual的但与该虚函数有相同参数和函数返回值类型的同名函数

在什么情况下应当声明虚函数：

  1. 成员函数所在类会作为基类
  2. 成员函数在类被继承后可能会被更改
  3. 对成员函数的调用是通过基类指针或引用去访问的
  4. 有时只在基类声明一个虚函数，函数体是空的，为了在派生中方便添加

说明：

  1. 使用虚函数，系统需要有一定的空间开销
  2. 一个类有虚函数时，编译系统会为该类构造一个虚函数表，是一个指针数组，存放每个虚函数的地址

# 命名空间 namespace

避免全局名字冲突
制定一些有名字的空间域，把一些全局实体分别放在各个命名空间中，从而与其他全局实体分开

// 定义
namespace 命名空间名{
    变量（可以有初始化）
    常量
    函数（可以定义或声明）
    结构体
    类
    模板
    命名空间
}
// 使用
命名空间名::命名空间成员名;
// 1. 别名
namespace 别名 = 命名空间名;
别名::命名空间成员名;
// 2. using 命名空间成员名
using 命名空间名::命名空间成员名;
命名空间成员名;
// 3. using namespace 命名空间名
using namespace 命名空间名;
命名空间成员名;
