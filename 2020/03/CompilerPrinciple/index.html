# 绪论

# 面向机器和面向人类的语言

分类 示例

汇编语言 Add si, ax

# 语言之间的翻译


高级语言间的翻译被称为转换或预处理
高级语言翻译成机器语言或汇编语言称为编译
从汇编语言到机器语言的翻译称为汇编
将一个汇编语言程序汇编为可在另一机器上运行的机器指令称为交叉汇编
建立在交叉汇编基础上的编译模式称为交叉编译
把机器语言或汇编语言翻译成汇编语言称为反汇编或反编译
翻译软件称为转换器、编译器

# 编译器和解释器

# 编译器

将源代码转换成可执行程序，然后再运行可执行程序

VC、delphi


源程序和数据是在不同时间（即分别在编译阶段和运行阶段）进行处理的
编译时间   实现源程序到目标程序的转换所占用的时间
执行时间   目标程序运行所占用的时间

# 解释器

在程序界面中输入一条命令后，将直接得到运行结果

Matlab、foxpro、数据库的查询端


源程序和数据是在相同时间进行处理的，边解释边执行
解释程序解释执行源程序，但并不生成目标程序，可以看成是一台虚拟机

# 比较

编译器   以源程序输入顺序处理程序语句
解释器   按照源程序的逻辑流程进行工作

解释器具有较好的动态特性和较好的可移植性，但是在时间和空间上损失较大，运行效率低
时间   运行过程中，解释器要时间来检查源程序，甚至要花时间进行存储空间分配
空间   要占用用户程序的运行空间以及系统的内存空间

# 编译器的工作原理和基本组成

# 通用程序设计语言主要成分

通用程序设计语言的典型特征之一是抽象，分为三种形式

  • 过程   C、nginx
  • 模块（抽象数据类型、ADT、程序包）   Ada83
  • 类   C++、Java、Ada95

声明和操作是最为核心的语言成分，也是程序设计语言的最基本的两个构成元素

  • 声明性语句   提供操作对象的性质，如数据类型、值、作用域等
  • 操作性语句   确定操作的计算次序，完成实际操作

编译器对两类语句的翻译

  • 声明   生成相应的环境，一般是配置存储空间
  • 操作   生成可执行的代码序列

# 以阶段划分编译器


编译程序的工作一般可以分为五个阶段

  • 词法分析
  • 语法分析
  • 语义分析与中间代码产生
  • 优化
  • 目标代码产生

此外，还有两个贯穿整个编译过程的工作

  • 符号表管理
  • 错误处理

# 编译器各阶段的工作

# 词法分析 Lexical Analysis

功能：从左至右读源程序（字符流），识别单词符号（又称记号token）
源程序字符序列 → 单词符号序列
又称线性分析或扫描（Linear Analysis，Scanning）
token分为

  1. 关键字   var begin end等，也称为保留字，表示有特殊含义的单词
  2. 标识符   x y z sort等，表示变量名、过程名、类型名、标号等
  3. 字面量   60 university等，表示常量
  4. 特殊符号   := + ;等，表示有特殊含义的运算符、分隔符等
# 例子

position = initial + rate * 60

单词类型 单词值
标识符1(id1) position
算符(赋值) =
标识符2(id2) initial
算符(加) +
标识符3(id3) rate
算符(乘) *
整数 60

# 语法分析 Syntax Analysis

功能：层次分析（Hierarchical Analysis）
依据：源程序的语法规则
单词符号序列 → 分析树（Parse Tree）（或语法树-Syntax Tree）
语法分析又称解析（Parsing），语法树的数据结构一般为二叉树

语法树是分析树的压缩表示

# 例子

position = initial + rate * 60

# 语义分析 Semantic Analysis

功能：语义检查，即验证语法结构合法的程序是否在语义上正确（程序的各个组成部分组合在一起是否有意义）

  • 收集代码生成阶段需要的语义信息
  • 类型检查与类型转换

分析树 → 带语义（注释）的树

# 例子

position = initial + rate * 60

# 中间代码产生 Intermediate Code Generation

功能：生成源程序的中间表示
通过树的后序遍历，生成三地址代码（three address-code），是一种四元式
带语义（注释）的树 → 中间代码

树的叶子结点为操作数，根结点为操作码

# 例子

position = initial + rate * 60

# 代码优化 Code Optimization

对中间代码进行优化，减少临时变量，以提高目标程序的时间与空间效率，与目标机器无关

# 例子

# 目标代码产生 Code Generation

功能：生成目标代码
目标代码为机器代码（machine code）、可重定位的机器代码（relocatable machine code）、汇编语言代码（assembly code）

movf id3, R2
mulf #60.0, R2
movf id2, R1
addf R2, R1
movf R1, id1

# 符号表管理 Symbal-Table Management

功能：管理分析过程中得到的源程序中的标识符的各种信息

  • 记录源程序中使用的标识符（identifier）
  • 收集每个标识符的各种属性（attribute）信息，包括类型（type）、作用域（scope）、存储分配（storage allocated）信息

# 出错处理 Error Detection and Reporting

功能：

  • 检查错误的位置
  • 检查错误的性质（词法、语法、语义……）
  • 错误恢复

# 编译器的分析/综合模式


前端主要由与源语言有关而与目标机器无关的那些部分组成

  • 词法分析、语法分析、符号表的建立、语义分析和中间代码生成
  • 与机器无关的代码优化工作
  • 相应的错误处理工作和符号表操作

后端由编译程序中与目标机器有关的部分组成

  • 与机器有关的代码优化、目标代码的生成
  • 相应的错误处理和符号表操作

把编译程序划分成前端和后端的优点：
便于移植、便于编译程序的构造

# 编译器扫描的遍数

把每个阶段将程序完整分析一遍的工作模式称为一遍扫描，为了减少存储空间存放中间表示以及减少不必要的输入，编译器往往把若干个阶段工作结合起来，对应进行一遍扫描
除了一遍扫描之外，还有多遍扫描

# 一遍扫描的编译程序

# 多遍扫描的编译程序


优点：

  • 减少对主存容量的要求
  • 使各遍编译程序功能独立、单纯，相互联系简单，编译程序结构清晰
  • 实现更充分的优化工作，获得高质量目标程序
  • 分离前后端，为编译程序的移植创造条件

缺点：

  • 增加了不少重复性的工作

# 编译程序的前后处理器

# 预处理器

功能：

  • 宏处理
  • 文件包含
  • 语言扩充
# 宏处理

宏处理器处理两类语句，即宏定义和宏调用

  • 宏定义   通常用统一的字符或关键字表示，如define或macro，宏定义由宏名字及宏体组成，通常宏处理器允许在宏定义中使用形参
  • 宏调用   由调用宏的命令名（宏名）和所提供的实参组成。宏处理器用实参代替宏体中的形参，再用变换后的宏体替换宏调用本身
# 文件包含

把文件的包含声明扩展为程序正文，也就是用被包含文件中的内容替换文件包含语句

# 语言扩充

用更先进的控制流和数据结构来增强原来的语言

# 汇编程序

汇编语言用助记符表示操作码，用标识符表示存储地址
最简单的汇编程序对输入作两遍扫描

# 第一遍扫描

找出标志存贮单元的所有标识符，并将它们存贮到汇编符号表中
在符号表中指定该标识符所对应的存储单元地址，此地址是在首次遇到该标识符时确定的

# 第二遍扫描

把每个用助记符表示的操作码翻译为二进制表示的机器代码
把用标识符表示的存储地址翻译为汇编符号表中该标识符所对应的地址
输出通常是可重定位的机器代码，输出中要对那些需要重定位的指令做出标记
可重定位

  • 起始地址为0，各条指令及其所访问的地址都是相对于0的逻辑地址
  • 当装入内存时，可以指定任意的地址L作为开始单元

# 连接装配程序

两个主要任务：

  1. 装入
    • 读入可重定位的机器代码
    • 修改重定位地址
    • 把修改后的指令和数据放在内存的适当地方或形成可执行文件
  2. 连接
    • 把几个可重定位的机器代码文件连接成一个可执行的程序

# 编译器的编写过程

# 以汇编语言和机器语言为工具

优点   可以针对具体的机器，充分发挥计算机的系统功能，生成的程序效率高
缺点   程序难读、难写、易出错、难维护、生产的效率低

# 高级语言书写

优点   程序易读、易理解、容易维护、生产的效率高
缺点   难以充分发挥计算机的系统功能，生成的程序效率低
可以利用已有的某种语言的编译程序实现另一语言的编译程序，也可以把一种机器上的编译程序移植到另一种机器上，实现自展技术

# 记号的属性

记号的属性是指记号的特征或特性，反映特征或特性的值是属性值
记号影响语法分析的决策，属性影响记号的翻译
在词法分析阶段，对记号只能确定一种属性

  • 关键字   一字一种或全体一种，前一种方法较好
  • 标识符   一般统归一种，也可分为若干种
  • 常数   一般统归一种，也可按类型分种
  • 运算符   一符一种或一类一种
  • 分界符   一般用一符一种

# 词法分析器的作用与工作方式

源程序由单词组成，单词是最小的语义单位
词法分析器的功能：

  • 扫描源程序字符流
  • 按照源语言的词法规则识别出各类单词符号
  • 产生用于语法分析的记号序列

词法分析器的辅助功能

  • 跳过源程序中的注释和空白
  • 把错误信息和源程序联系起来（错误定位）
  • 宏预处理

# 词法分析器与语法分析器的关系

# 1. 词法分析器作为语法分析器的子程序


优点：

  • 避免了中间文件
  • 省去了取送符号的工作
  • 有利于提高编译器的效率
# 2. 词法分析器单独进行一遍扫描


输出放入一个中间文件，可以是磁盘文件或内存文件

# 3. 与语法分析器并行工作


词法分析程序与语法分析程序在同一遍中，以生产者和消费者的关系同步运行

# 输入缓冲区

通过设立输入缓冲区来加快读入源程序字符序列的速度
超前搜索   为了得到某一个单词符号的确切性质，需要超前扫描若干个字符
输入缓冲区一般被设计为一块与磁盘扇区大小成倍数关系的内存，保证对缓冲区的一次输入所需I/O操作尽可能少，一般采用单缓冲区和**双缓冲区（缓冲区对）**的方式

# 单缓冲区

一般使用两个指针标识当前扫描的位置，第一个指针表示当前被识别的记号的第一个字符，另一指针表示现在正在扫描的字符

# 字母表 alphabet

字母表是符号（symbols）的非空有穷集合，用Σ\SigmaΣ表示

# 符号串 string

符号串是由字母表中的符号所组成的有穷序列
在语言理论中，符号串又称为句子（sentence）、字（word）
符号串的长度为符号串中包含符号的个数，符号串sss的长度记为∣s∣|s|∣s∣

ε\varepsilonε表示空串，∣ε∣=0|\varepsilon|=0∣ε∣=0

# 子序列

从sss中删去0或若干个不一定连续的字符后得到的符号串

# 不确定性

NFA的特点是它的不确定性，即在当前状态下，对同一个字符ch，可能有多于一个的下一状态转移
反映在NFA的定义中，就是move函数是一对多的
反映在转换图中，就是从一个节点可通过多于一条标记相同字符的边转移到不同的状态
反映在转换矩阵中，就是M[si,sj]M[s_i, s_j]M[si​,sj​]中不是一个单一状态，而是一个状态的集合

# 识别输入序列

从NFA的初态开始，对于输入序列中的每一个字符，寻找它的下一状态转移，直到没有下一状态转移为止
若此时所处状态是终态，则从初态到终态路径上的所有标记，构成了一个识别出的记号。否则沿原路返回，并在返回的每一个节点试探可能的下一条路径，直到遇到第一个终态，或者一直返回到初态也没有遇到终态
对于输入序列，若试探了所有的路径也找不到下一状态转移或不能到达一个终态，则NFA不接受该序列，说明它不是语言中的合法记号。若到达一个终态，则NFA接受该序列，说明它是语言中的一个合法记号

# 缺点

  1. 只有尝试了全部可能的路径，才能确定一个输入序列不被接受，而这些路径的条数随着路径长度的增长成指数增长
  2. 识别过程中需要进行大量的回溯，时间复杂度与输入序列成指数级增长，且算法复杂

            int move(int state, char inChar) {
                switch(state) {
                    case 0:
                        if (inChar == 'a') return 1;
                        if (inChar == 'b') return 0;
                        break;
                    case 1:
                        if (inChar == 'a') return 1;
                        if (inChar == 'b') return 2;
                        break;
                    case 2:
                        if (inChar == 'a') return 1;
                        if (inChar == 'b') return 3;
                        break;
                    case 3:
                        if (inChar == 'a') return 1;
                        if (inChar == 'b') return 0;
                        break;
                }
                return 0;
            }
            // 判断 state 是否为终结状态
            bool isFinished(int state) {
                if (state == 3) return true;
                return false;
            }

            while(curChar!=inStr.Length) {
                currrentState = move(currrentState, inStr[curChar]);
                curChar++;
            }

                Console.WriteLine("yes!");
            }
            else {
                Console.WriteLine("No!");
            }
        }

左部非终结符相同的产生式合并成一个产生式，并以此非终结符为该产生式命名，所有产生式的右部由符号∣|∣连接，每个右部为产生式的一个候选项，每个候选项具有平等的权力

# CFG产生语言的基本方法——推导

  1. 直接推导   根据A→γA\to\gammaA→γ 得到αAβ→αγβ\alpha A\beta \to \alpha \gamma \betaαAβ→αγβ
  2. 多步推导   任意文法序列α1,α2,…αn\alpha_1,\alpha_2,\dots \alpha_nα1​,α2​,…αn​，有α1  ⟹  α2  ⟹  ⋯  ⟹  αn\alpha_1\implies \alpha_2\implies \dots \implies \alpha_nα1​⟹α2​⟹⋯⟹αn​，记为α1⇒∗αn\alpha_1 \xRightarrow{*} \alpha_nα1​∗​αn​。
  3. 零步推导   α1=αn\alpha_1=\alpha_nα1​=αn​时
  4. 至少一步推导   α1≠αn\alpha_1\not =\alpha_nα1​​=αn​时，记为α1⇒+αn\alpha_1 \xRightarrow{+} \alpha_nα1​+​αn​

# 上下文无关语言 CFL

L(G)={ω∣S⇒+ωandω∈T∗}\begin{aligned} L(G)=\{\omega|S \xRightarrow{+} \omega and \omega \in T^*\} \end{aligned} L(G)={ω∣S+​ωandω∈T∗}​

其中，ω\omegaω为句子
若S⇒+α,α∈(N∪T)∗S \xRightarrow{+} \alpha ,\alpha \in (N\cup T)^*S+​α,α∈(N∪T)∗，则称α\alphaα为GGG的一个句型

在推导过程中，若每次推导都替换最左边的非终结符，则称为最左推导，产生的句型为左句型；若每次推导都替换最右边的非终结符，则称为最右推导，产生的句型为右句型

最右推导被称为规范推导

# 分析树

对CFG GGG的句型，分析树被定义为具有下列性质的一棵树

  1. 根由开始符号SSS所标记
  2. 每个叶子由一个终结符、非终结符或ε\varepsilonε标记
  3. 每个内部节点由一个非终结符标记
  4. 若A→X1X2…XnA\to X_1X_2\dots X_nA→X1​X2​…Xn​是一个产生式，则X1,X2,…,XnX_1,X_2,\dots,X_nX1​,X2​,…,Xn​是标记为AAA的内部节点从左到右所有孩子的标记。若A→εA\to \varepsilonA→ε，则标记为AAA的节点可以仅有一个标记为ε\varepsilonε的孩子

# 分析树与文法和语言的关系

  1. 每一推导或每个产生式对应一棵仅有父子关系的子树，即产生式左部非终结符长出右部的孩子
  2. 分析树的叶子从左到右构成GGG的一个句型。若叶子仅由终结符标记，则构成一个句子

每个高度为2的树为一个产生式

# 语法树

对CFG GGG的句型，语法树被定义为具有下列性质的一棵树

  1. 根与内部节点由表达式中的操作符标记
  2. 叶子由表达式中的操作数标记
  3. 用于改变运算优先级和结合性的括弧被隐含在语法树的结构中

# 递归下降子程序

为每个非终结符构造一个子程序，每个子程序的过程体按改产生式的候选项分情况展开，遇到终结符直接匹配，遇到非终结符调用相应非终结符的子程序。

# 预测分析器

数学模型是下推自动机

  1. 构造预测分析表
  2. 给定串进行预测

预测分析表
行是非终结符，为文法产生式的左部
列是终结符，列中要再加一个终结符#表示记号流的开始和结束

First集

  1. 对于右部第一个符号是终结符的可直接确定
  2. 不含的递推第一个符号就可以得到相应的终结符
  3. 有ε\varepsilonε暂时不管，但是ε\varepsilonε与First集无关，只与Follow集有关

Follow集
将非终结符变为ε\varepsilonε，保证#S#可以转换为##
求X的Follow集

  1. 左部为X的文法产生式最右部均为X，则X的Follow集为自己，然后找文法产生式右部有X的产生式，X的Follow集就为这些产生式左部的Follow集的并集（递归）
  2. 后面跟非终结符，X的Follow集为非终结符的First集。如果非终结符的First集有ε\varepsilonε，则非终结符的First集要合并进X的Follow集
  3. 后面跟的是终结符，则X的Follow集就为终结符组成的集合
  4. 对于开始符号，Follow集要加上#

用当前字符和栈顶元素去查预测分析表，当匹配到了当前字符，就匹配下一个字符
最后栈内剩下#且当前输入仅剩#的时候表示语法正确，否则错误

相当于树的后根遍历，所以压栈的时候要从右往左压

# 移进归约分析表 SLR

行：状态，列：终结符 非终结符（goto表）
s id：移进，id号文法产生式
r id：规约，弹出id号文法产生式，压入非终结符+状态

根据文法产生SLR：

  1. 得到活前缀DFA
  2. 如果 状态1，终结符 -> 状态2，表[状态1，终结符]填 s状态2
  3. 如果 状态1，非终结符 -> 状态2,表[状态1，非终结符]填 状态2（goto表）
  4. 对于状态集中点在最右边的产生式，求左部的Follow集（不要忘了#），表[状态，Follow集]填 r点之前的产生式编号
  5. 如果某状态集中点之前的是拓广文法，在表[状态，#]交叉点填 acc

# 符号表条目

每个声明的名字在符号表中占据一栏，称为一个条目
指向这些信息的指针放在符号表条目的相应域(field，也被称为字段)中
符号表中的内容可以包括保留字、标识符、特殊符号(包括算符、分隔符等)等等
标识符是最大的一类符号
基本内容可以分为名字和属性
一个名字的组合关键字至少应该包括三项：名字＋作用域＋类型

# 构成名字的字符串

如果把名字的字符串信息本身直接存放在符号表中，则会造成很大浪费，同时也无法解决极端情况
直接将构成名字的字符串存放在符号表条目中的方式被称为直接存储方式
间接存储，就是将构成名字的字符串统一存放在一个大的连续空间中，字符串与字符串之间用特殊的分隔符隔开，而在符号表的条目中，仅存放指向该字符串首字符的指针即可，访问时间接寻址

# 名字的作用域

一个名字在哪个范围内起作用，被称为名字的作用域
分别在并列的两个范围内的名字作用域互不相干，但是分别在嵌套的两个范围内的名字，其作用域的问题就需要制定规则来限定，以使得任何一个名字在任何范围内涵义都是无二义的

  1. 静态作用域原则(static-scope rule)：编译时就可以确定名字的作用域，也可以说，仅从静态读程序就可确定名字的作用域
  2. 最近嵌套原则(most closely nested)（下边的作用域规则以程序块为例，但也适用于过程）:
    • 程序块B中声明的作用域包括B；
    • 如果名字x不在B中声明，那么B中x的出现是在外围程序块B’的x声明的作用域中，使得B’有x的声明，并且B’比其它任何含x声明的程序块更接近被嵌套的B

名字的声明在离其最近的内层起作用，即在名字引用处从内向外看，它处在所遇到的第一个该名字声明的作用域

引入栈
对于任何一个名字，从栈顶开始向栈底查找，遇到的第一个符合条件的名字，即是所要查找的符号。当要插入一个名字的时候，也是首先在符号表中查找：若查到，则返回该名字在符号表中的位置(指针或下标)；否则加入到栈顶
当从某个作用域退出时，从栈顶把该作用域的所有名字全部摘走，存放在一个不活动的临时表中，以备后用
