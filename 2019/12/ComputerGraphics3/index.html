中点Bresenham画直线

直线方程

F(x,y)=y−kx−b=0\begin{aligned} F(x,y)=y-kx-b=0 \end{aligned}​F(x,y)=y−kx−b=0​​

直线将平面分成三个区域

  1. F(x,y)=0F(x,y)=0F(x,y)=0，点在直线上
  2. F(x,y)>0F(x,y) \gt 0F(x,y)>0，点在直线上方
  3. F(x,y)<0F(x,y) \lt 0F(x,y)<0，点在直线下方

原理
每次在最大位移方向走一步，另一个方向是否走步取决于误差项的判别（判断中点与直线的位置关系）

  1. di<0d_i \lt 0d​i​​<0 di+1=F(xi+2,yi+1.5)=di+1−k=di+2Δx−2Δy\begin{aligned} d_{i+1}&=F(x_i+2,y_i+1.5) \\ &=d_i+1-k \\ &=d_i+2\Delta x-2\Delta y \end{aligned}​d​i+1​​​​​​​=F(x​i​​+2,y​i​​+1.5)​=d​i​​+1−k​=d​i​​+2Δx−2Δy​​

代码

void MidBresenham(int dx, int dy, int x, int y, int end){
    int d=dx-2*dy; // 初始值
    while(x<=end){
        glVertex2i(x, y); // 画点
        x++;
        if(d<0){ // d<0, 取上面点
            y++;
            d+=(2*dx-2*dy);
        }
        else d+=(-2*dy);
    }
}

代码

void Bresenham(int dx, int dy, int x, int y, int end){
    int d=-dx; // 初始值
    while(x<=end){
        glVertex2i(x, y); // 画点
        d+=2*dy;
        if(d>=0){ // d>=0，选择上面的点
            y++;
            d+=(-2*dx);
        }
        x++;
    }
}

void MidBresenhamCircle(int r){
    int x, y, d;
    x=0; y=r; d=1-r; // 初始值

多边形的扫描转换

点阵表示

x扫描线算法

算法步骤

  1. 确定多边形顶点的最小和最大yyy值（yminy_{min}y​min​​和ymaxy_{max}y​max​​），得到多边形所占有的最大扫描线数
  2. 从y=yminy=y_{min}y=y​min​​到y=ymaxy=y_{max}y=y​max​​，每次用一条扫描线进行填充
    对一条扫描线填充的过程可分为4个步骤：
    ① 求交   计算扫描线与多边形所有边的交点
    ② 排序   所有交点按xxx坐标递增顺序排序
    ③ 交点配对   配对第1/2个、第3/4个交点……每对交点都代表扫描线与多边形的一个相交区间
    ④ 区间填色   相交区间内的像素置成填充色

避免多边形区域扩大化
左闭右开（左取右不取）
下闭上开（下取上不取）

顶点个数取值
检查顶点的两条边的另外两个端点的yyy值，两个yyy值中大于交点yyy值的个数是0、1、2，来决定取0、1、2个交点

缺点
需要与多边形所有的边求交，效率很低

改进的有效边表算法

也称yyy连贯性算法
有效边AE
指与当前扫描线相交的多边形的边，也称为活性边
有效边表AET
把有效边按与扫描线交点xxx坐标递增的顺序存放在一个链表中，也称为活性边表
有效边表的每个结点存放的对应边

边表ET
存放多边形除水平边外的所有边的信息
边表的构造步骤

  1. 首先构造一个纵向链表，链表的长度为多边形所占有的大扫描线数，链表的每个结点，称为一个桶，对应多边形覆盖的每一条扫描线
  2. 将每条边的信息装入与该边最小yyy坐标yminy_{min}y​min​​相对应的桶中。也就是说，若某边的较低端点为yminy_{min}y​min​​，则该边就放在相应的y=yminy=y_{min}y=y​min​​扫描线桶中
  3. 每条边的数据形成一个结点，内容包括：该扫描线与该边的初始交点xxx（即较低端点的xxx值），该边的最大yyy坐标值ymaxy_{max}y​max​​，以及斜边的倒数1/k1/k1/k
  4. 同一桶中若干条边按x∣yminx|y_{min}x∣y​min​​由小到大排序，若x∣yminx|y_{min}x∣y​min​​相等，则按照1/k1/k1/k由小到大排序


缩短边的策略

算法步骤

  1. 初始化。构造边表，AET表置空
  2. 将第一个不空的ET表中的边与AET表合并
  3. 由AET表中取出交点并进行填充
    设置一个bool变量b=falseb=falseb=false，指针从AET第一个结点到最后一个结点遍历一次，每访问一个结点把bbb取反
    若bbb为真，则把从当前结点的xxx值开始到下一个结点的xxx值结束的区间置成填充色，填充之后删除y=ymaxy=y_{max}y=y​max​​的边

    需要避免多边形区域的扩大化，左闭右开，下闭上开

  4. yi+1=yi+1y_{i+1}=y_i+1y​i+1​​=y​i​​+1，根据xi+1=xi+1/kx_{i+1}=x_i+1/kx​i+1​​=x​i​​+1/k计算并修改AET表，同时合并ET表中y=yi+1y=y_{i+1}y=y​i+1​​桶中的边，按次序插入到AET表中，形成新的AET表
  5. AET表不为空则转第3步，否则结束

边标志算法

基本思想
先用一种特殊的颜色在帧缓冲存储器中将多边形的边界（水平边除外）勾画出来，然后将着色的边界点依xxx坐标递增的顺序两两配对，再将每对像素构成的扫描线区间内的所有像素置为填充色
步骤

  1. 打标记   对每条边进行直线扫描转换并在得到的像素点上打标记

  2. 填充   按左闭右开原则进行填色
    设置一个bool变量insideinsideinside指示当前点状态
    若点在多边形内，则insideinsideinside为真
    若点在多边形外，则insideinsideinside为假

    insideinsideinside初值为假

    每当当前访问像素为已打上标志的点，就insideinsideinside取反
    对未打标志的点insideinsideinside不变
    若处理后insideinsideinside为真，则置该像素为填充色


代码

// 多边形坐标与标志数组
const int pointsx[7]={3,6,8,12,7,3,1};
const int pointsy[7]={1,5,1,9,8,12,7};
bool flags[14][14]={false};
// 标志
void mark(){
    for(int i=0;i<=6;i++){
        int next=(i+1)%7, pre=(i+7-1)%7;
        MidBresenhamLine(pointsx[i], pointsy[i], pointsx[next], pointsy[next]); // 中点Bresenham直线扫描
        // 特判交点
        if((pointsy[pre]<pointsy[i]&&pointsy[next]>pointsy[i])||(pointsy[pre]>pointsy[i]&&pointsy[next]<pointsy[i])){
            flags[pointsx[i]][pointsy[i]]=!flags[pointsx[i]][pointsy[i]];
        }
    }
}
// 填充
void fill(){
    for(int j=13;j>=0;j--){
        bool inside=false;
        for(int i=0;i<=13;i++){
            if(flags[i][j])inside=!inside; // inside取反
            if(inside)glVertex2i(i, j); // 画点
        }
    }
}
// 边标志填充
void EdgeMarkFill(){
    mark(); // 标志
    fill(); // 填充
}

区域填充

区域
指已经表示成点阵形式的填充图形，是一个像素的集合
内点表示
枚举出给定区域内部的所有像素，要求区域内的所有像素着同一颜色，区域外的像素着不同的颜色
边界表示
枚举出边界上所有的像素，边界的上所 有像素着同一颜色，内部像素着与边界像素 不同的颜色，边界包围的是区域本身

区域填充
从区域的一点（种子点）开始，由内向外将填充色扩展到整个区域的过程
要求区域是连通的
4-邻接点
上、下、左、右4个相邻的点
8-邻接点
上、下、左、右、左上、右上、左下、右下8个相邻的点
4-连通区域
从区域上一点出发，通过访问已知点的4-邻接点，在不越出区域的前提下，遍历区域内的所有像素
8-连通区域
通过访问区域内已知点的8-邻接点来遍历整个区域
4-连通区域与8-连通区域的主要区别

  1. 邻接点不同   如果能通过4-邻接点完全遍历，就能通过8-邻接点完全遍历；反之不成立
  2. 边界要求不同   8-连通区域的边界要求比4-连通区域的边界要求高

边界填充算法

以边界表示法为基础的区域填充算法
4-连通边界填充算法

  1. 种子像素入栈
  2. 执行如下三步操作
    ① 栈顶像素出栈
    ② 将出栈像素置成填充色
    ③ 检查出栈像素的4-邻接点，若其中某个像素点不是边界色且未置成多边形色，则把该像素入栈
  3. 检查栈是否为空，若栈非空重复执行上一步骤，否则结束

沿扫描线的4-连通边界填充算法

  1. 种子像素入栈
  2. 执行如下三步操作
    ① 栈顶像素出栈
    ② 填充出栈像素所在扫描行的连续像素段，从出栈的像素开始沿扫描线向左和向右填充，直到遇到边界像素为止，即每出栈一个像素，就对包含该像素的整个扫描线区间进行填充，并且记录下此时扫描线区间的xxx坐标范围[x1,x2][x_1,x_2][x​1​​,x​2​​]
    ③ 分别检查上、下两条扫描线上位于[x1,x2][x_1,x_2][x​1​​,x​2​​]坐标区间内的未被填充的连续水平像素段，将每个连续像素段的最左像素取作种子像素压入堆栈
  3. 检查栈是否为空，若栈非空重复执行上一步骤，否则结束

泛填充算法

以内点表示法为基础的区域填充算法
4-连通泛填充算法

  1. 种子像素入栈
  2. 执行如下三步操作
    ① 栈顶像素出栈
    ② 将出栈像素置成填充色
    ③ 检查出栈像素的4-邻接点，若其中某个像素点是给定内部点的颜色，则把该像素入栈
  3. 检查栈是否为空，若栈非空重复执行上一步骤，否则结束

沿扫描线的4-连通泛填充算法

  1. 种子像素入栈
  2. 执行如下三步操作
    ① 栈顶像素出栈
    ② 填充出栈像素所在扫描行的连续像素段，从出栈的像素开始沿扫描线向左和向右填充，直到遇到边界像素为止，即每出栈一个像素，就对包含该像素的整个扫描线区间进行填充，并且记录下此时扫描线区间的xxx坐标范围[x1,x2][x_1,x_2][x​1​​,x​2​​]
    ③ 分别检查上、下两条扫描线上位于[x1,x2][x_1,x_2][x​1​​,x​2​​]坐标区间内的未被填充为新的填充色的连续水平像素段，将每个连续像素段的最左像素取作种子像素压入堆栈
  3. 检查栈是否为空，若栈非空重复执行上一步骤，否则结束

内-外测试

奇-偶规则
从任意位置p作一条射线，若与该射线相交的多边形边的数目为奇数，则p是多边形内部点，否则是外部点
非零环绕数规则

  1. 按逆时针方向使多边形的边变为矢量
  2. 将环绕数初始化为零
  3. 从测试区域的任意位置p作一条射线，该射线不与任何多边形顶点相交。当从p点沿射线方向移动时，对在每个方向上穿过射线的边计数
    每当多边形的边从右到左穿过射线时，环绕数加1，从左到右时，环绕数减1
  4. 处理完多边形的所有相关边之后，若环绕数为非零，则p为内部点，否则，p是外部点
